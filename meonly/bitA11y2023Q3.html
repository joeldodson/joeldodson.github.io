  1. Page Title: Request a free demo | Udemy Business
  2. Page URL: https://business.udemy.com/request-demo-mx?locale=en_US&user_type=visitor&utm_type=mx "
  3. Success Criteria - Failed: Name, Role, Value
  4. Success Criteria Identifier: 4.1.2
  5. Description: There are a set of "buttons" on the Udemy for Business page. I'll focus on the "What we do" component.

    The "What we do" button acts like a button. NVDA reads out "button" when I navigate to it. Single letter navigation 'b' lands on the component. But the space and enter keys have no effect.

    The code when I reveal source (ctrl+u in Chrome) shows the following element: 

        <a href="javascript:void(0);" class="section-header disabled" aria-haspopup="true" aria-expanded="false" aria-label="What we do" role="button" tabindex="0">What we do</a>
    What the heck is all that?

    We're starting with an anchor element to implement a button. The href of "javascript:void(0);" causes the link to do nothing (according to the folowing Stack Overflow article: https://stackoverflow.com/questions/1291942/what-does-javascriptvoid0-mean). And there is a role="button" and some other ARIA Iincluding a redundant label and tabstop.

    I'd argue this not only failes 4.1.2, it also fails 2.1.1 and fails the idea of "no ARIA is better than bad ARIA."
